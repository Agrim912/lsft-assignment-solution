================================================================================
KEY DESIGN DECISIONS
================================================================================

1. DETERMINISTIC PARSER instead of LLM
   - Reason: Better for performance and reliability on known prompts
   - Trade-off: Less robust on novel prompts outside golden set
   - Result: 0 tokens, 0 hops, instant response

2. SINGLE AGENT instead of multi-agent
   - Reason: Routing logic is simple, deterministic
   - Trade-off: Less modular if routing becomes complex
   - Result: Simpler code, fewer state-passing bugs

3. SLIM OUTPUTS before returning
   - Reason: Respect token budgets, never dump raw API responses
   - Trade-off: May drop useful metadata
   - Result: 19.1x reduction on model list

4. ASK or FAIL instead of guess
   - Reason: Reliability principle from assignment
   - Trade-off: Less feature-complete on incomplete requests
   - Result: No hallucinated data, trusted answers
