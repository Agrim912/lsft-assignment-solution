================================================================================
KEY PARSING RULES
================================================================================

1. ACTION DETECTION (Priority order prevents conflicts)
   - "list" + "model" → list_models
   - "locked" + "channel" → locked_channels
   - "current budget" → get_current_budget
   - "compare" + ("vs"|"versus"|"which wins") → compare_scenarios
   - "how much budget" + ("hit"|"reach") → target_kpi
   - "optimise"|"optimize" → optimise
   - ("what revenue"|"would deliver") + ("model"|"quarter") → forecast
   - "conversions" or "installs" → ambiguous
   - else → missing_input

2. AMOUNT EXTRACTION
   Regex: r"\$\s*([0-9,]+(?:\.[0-9]+)?)\s*([kKmM])?"
   
   Examples:
   - "$500k" → 500000.0 (k = thousands)
   - "$1M" → 1000000.0 (M = millions)
   - "$1,000,000" → 1000000.0
   - "$ 500k" → 500000.0 (handles space)

3. CONSTRAINT TYPE MAPPING
   - "aggressive" → Aggressive → 3
   - "moderate" → Moderate → 2
   - "conservative" → Conservative → 1
   - "current" + "constraints" → Current → 0

4. MODEL ID RESOLUTION
   - Explicit ID (numeric): Use as-is
   - "latest"|"recent"|"most recent": Keep as "latest" (resolved by handler)
   - Missing: Set to None

5. VALIDATION (Changes action to 'missing_input' if)
   - optimise: no model OR no budget
   - forecast: no model OR no budget
   - target_kpi: no model OR no target_revenue
   - compare_scenarios: < 2 amounts
   - unknown action: always becomes missing_input
